1 DOCKER 启动

启动 MinIO 容器

你已经拉取了镜像，现在用以下命令启动容器。数据会持久化到本地 D:\minio\data 目录。

bash 
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e "MINIO_ROOT_USER=minioadmin" \
  -e "MINIO_ROOT_PASSWORD=minioadmin" \
  -v D:\minio\data:/data \
  minio/minio server /data --console-address ":9001"
💡 说明

9000 端口 → API 接口（你的 Java 程序连这个）

9001 端口 → Web 管理控制台（浏览器访问）

启动后访问 http://localhost:9001，用 minioadmin / minioadmin 登录控制台。

登录控制台后，手动创建一个 Bucket（存储桶），例如命名为 weixiu。你也可以不手动创建，后面的代码会自动创建。

2 MAVEN 依赖

添加 MinIO SDK 依赖

在你项目的 pom.xml 中添加以下依赖：

xml pom.xml 
<!-- MinIO Java SDK -->
<dependency>
    <groupId>io.minio</groupId>
    <artifactId>minio</artifactId>
    <version>8.5.7</version>
</dependency>
⚠️ 依赖冲突

如果你的项目使用了 OkHttp 或 Kotlin 相关依赖，MinIO SDK 可能产生版本冲突。出现问题时在 <dependency> 中添加以下排除：

xml 冲突时的排除写法 
<dependency>
    <groupId>io.minio</groupId>
    <artifactId>minio</artifactId>
    <version>8.5.7</version>
    <exclusions>
        <exclusion>
            <groupId>com.squareup.okhttp3</groupId>
            <artifactId>okhttp</artifactId>
        </exclusion>
    </exclusions>
</dependency>
3 配置文件

application.yml 配置

在 application.yml（或 .properties）中添加 MinIO 连接信息：

yaml application.yml

  endpoint: http://localhost:9000
  access-key: minioadmin
  secret-key: minioadmin

4 配置类

MinIO 配置类 & 属性类

创建两个类：一个读取配置属性，一个注册 MinioClient Bean。

📂 src/main/java/com/yourpackage/
  ├── 📂 config/
  │   ├── MinioProperties.java
  │   └── MinioConfig.java
  ├── 📂 service/
  │   └── MinioService.java
  └── 📂 controller/
      └── FileController.java
java MinioProperties.java 
package com.yourpackage.config;

import lombok.Data;
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

@Data
@Component
@ConfigurationProperties(prefix = "minio")
public class MinioProperties {

    /** MinIO 服务地址，例如 http://localhost:9000 */
    private String endpoint;

    /** 访问密钥 */
    private String accessKey;

    /** 秘密密钥 */
    private String secretKey;

    /** 默认存储桶名称 */
    private String bucket;
}
💡 没用 Lombok？

如果你的项目没引入 Lombok，把 @Data 去掉，手动写 getter / setter 即可。

java MinioConfig.java 
package com.yourpackage.config;

import io.minio.MinioClient;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class MinioConfig {

    @Bean
    public MinioClient minioClient(MinioProperties props) {
        return MinioClient.builder()
                .endpoint(props.getEndpoint())
                .credentials(props.getAccessKey(), props.getSecretKey())
                .build();
    }
}
5 SERVICE 层

MinioService —— 核心操作封装

这是最关键的一个类，封装了所有常用操作：上传、下载、删除、获取预签名URL、检查桶是否存在等。

java MinioService.java 
package com.yourpackage.service;

import com.yourpackage.config.MinioProperties;
import io.minio.*;
import io.minio.http.Method;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.web.multipart.MultipartFile;

import javax.annotation.PostConstruct;
import java.io.InputStream;
import java.util.UUID;
import java.util.concurrent.TimeUnit;

@Slf4j
@Service
@RequiredArgsConstructor
public class MinioService {

    private final MinioClient minioClient;
    private final MinioProperties minioProperties;

    /**
     * 项目启动时自动检查并创建 Bucket
     */
    @PostConstruct
    public void init() {
        try {
            boolean exists = minioClient.bucketExists(
                    BucketExistsArgs.builder()
                            .bucket(minioProperties.getBucket())
                            .build()
            );
            if (!exists) {
                minioClient.makeBucket(
                        MakeBucketArgs.builder()
                                .bucket(minioProperties.getBucket())
                                .build()
                );
                log.info("Bucket [{}] 创建成功", minioProperties.getBucket());
            }
        } catch (Exception e) {
            log.error("初始化 MinIO Bucket 失败: {}", e.getMessage());
            throw new RuntimeException("MinIO 初始化失败", e);
        }
    }

    /**
     * 上传文件
     * @param file Spring 的 MultipartFile
     * @return 文件在 MinIO 中的对象名（objectName）
     */
    public String upload(MultipartFile file) {
        try {
            // 生成唯一文件名，避免覆盖
            String originalFilename = file.getOriginalFilename();
            String ext = "";
            if (originalFilename != null && originalFilename.contains(".")) {
                ext = originalFilename.substring(originalFilename.lastIndexOf("."));
            }
            String objectName = UUID.randomUUID().toString().replace("-", "") + ext;

            minioClient.putObject(
                    PutObjectArgs.builder()
                            .bucket(minioProperties.getBucket())
                            .object(objectName)
                            .stream(file.getInputStream(), file.getSize(), -1)
                            .contentType(file.getContentType())
                            .build()
            );

            log.info("文件上传成功: {}", objectName);
            return objectName;

        } catch (Exception e) {
            log.error("文件上传失败: {}", e.getMessage());
            throw new RuntimeException("文件上传失败", e);
        }
    }

    /**
     * 下载文件，返回输入流
     */
    public InputStream download(String objectName) {
        try {
            return minioClient.getObject(
                    GetObjectArgs.builder()
                            .bucket(minioProperties.getBucket())
                            .object(objectName)
                            .build()
            );
        } catch (Exception e) {
            log.error("文件下载失败: {}", e.getMessage());
            throw new RuntimeException("文件下载失败", e);
        }
    }

    /**
     * 删除文件
     */
    public void delete(String objectName) {
        try {
            minioClient.removeObject(
                    RemoveObjectArgs.builder()
                            .bucket(minioProperties.getBucket())
                            .object(objectName)
                            .build()
            );
            log.info("文件删除成功: {}", objectName);
        } catch (Exception e) {
            log.error("文件删除失败: {}", e.getMessage());
            throw new RuntimeException("文件删除失败", e);
        }
    }

    /**
     * 获取文件的预签名访问 URL（临时可访问链接）
     * @param objectName 对象名
     * @param expiry     过期时间（分钟）
     */
    public String getPresignedUrl(String objectName, int expiry) {
        try {
            return minioClient.getPresignedObjectUrl(
                    GetPresignedObjectUrlArgs.builder()
                            .method(Method.GET)
                            .bucket(minioProperties.getBucket())
                            .object(objectName)
                            .expiry(expiry, TimeUnit.MINUTES)
                            .build()
            );
        } catch (Exception e) {
            log.error("获取预签名 URL 失败: {}", e.getMessage());
            throw new RuntimeException("获取预签名 URL 失败", e);
        }
    }

    /**
     * 获取文件的永久访问地址（需要 Bucket 设置为 public）
     */
    public String getFileUrl(String objectName) {
        return minioProperties.getEndpoint() + "/"
                + minioProperties.getBucket() + "/" + objectName;
    }
}
⚠️ Spring Boot 3.x 用户注意

如果你用的是 Spring Boot 3.x + JDK 17，请把代码中的 javax.annotation.PostConstruct 替换为 jakarta.annotation.PostConstruct。

💡 没用 Lombok？

去掉 @Slf4j 和 @RequiredArgsConstructor，手动添加构造方法注入和 Logger：

java 不用 Lombok 的写法 
private static final Logger log = LoggerFactory.getLogger(MinioService.class);

private final MinioClient minioClient;
private final MinioProperties minioProperties;

public MinioService(MinioClient minioClient, MinioProperties minioProperties) {
    this.minioClient = minioClient;
    this.minioProperties = minioProperties;
}
6 CONTROLLER 层

FileController —— 提供 REST 接口

java FileController.java 
package com.yourpackage.controller;

import com.yourpackage.service.MinioService;
import lombok.RequiredArgsConstructor;
import org.springframework.http.HttpHeaders;
import org.springframework.http.MediaType;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.multipart.MultipartFile;

import java.io.InputStream;
import java.util.HashMap;
import java.util.Map;

@RestController
@RequestMapping("/api/file")
@RequiredArgsConstructor
public class FileController {

    private final MinioService minioService;

    /**
     * 上传文件
     * POST /api/file/upload
     */
    @PostMapping("/upload")
    public Map<String, Object> upload(@RequestParam("file") MultipartFile file) {
        String objectName = minioService.upload(file);
        String url = minioService.getFileUrl(objectName);

        Map<String, Object> result = new HashMap<>();
        result.put("code", 200);
        result.put("msg", "上传成功");
        result.put("objectName", objectName);
        result.put("url", url);
        return result;
    }

    /**
     * 下载文件
     * GET /api/file/download?objectName=xxx.jpg
     */
    @GetMapping("/download")
    public ResponseEntity<byte[]> download(@RequestParam String objectName) {
        try (InputStream is = minioService.download(objectName)) {
            byte[] bytes = is.readAllBytes();
            return ResponseEntity.ok()
                    .header(HttpHeaders.CONTENT_DISPOSITION,
                            "attachment; filename=\"" + objectName + "\"")
                    .contentType(MediaType.APPLICATION_OCTET_STREAM)
                    .body(bytes);
        } catch (Exception e) {
            return ResponseEntity.internalServerError().build();
        }
    }

    /**
     * 删除文件
     * DELETE /api/file/delete?objectName=xxx.jpg
     */
    @DeleteMapping("/delete")
    public Map<String, Object> delete(@RequestParam String objectName) {
        minioService.delete(objectName);

        Map<String, Object> result = new HashMap<>();
        result.put("code", 200);
        result.put("msg", "删除成功");
        return result;
    }

    /**
     * 获取临时访问链接
     * GET /api/file/presigned?objectName=xxx.jpg&expiry=60
     */
    @GetMapping("/presigned")
    public Map<String, Object> getPresignedUrl(
            @RequestParam String objectName,
            @RequestParam(defaultValue = "60") int expiry) {

        String url = minioService.getPresignedUrl(objectName, expiry);

        Map<String, Object> result = new HashMap<>();
        result.put("code", 200);
        result.put("url", url);
        return result;
    }
}

接口汇总：

方法 路径 参数 说明
POST /api/file/upload file (form-data) 上传文件，返回 objectName 和 url
GET /api/file/download objectName 下载文件流
DELETE /api/file/delete objectName 删除文件
GET /api/file/presigned objectName, expiry 获取临时访问链接

8 常见问题

FAQ & 排错

❌ 连接被拒绝 Connection refused

确认 Docker 容器正在运行：docker ps 查看 minio 容器状态。

确认端口 9000 没有被其他程序占用。

如果你的 Docker 跑在 WSL2 中，endpoint 可能需要改成 http://host.docker.internal:9000 或 WSL 的实际 IP。

❌ Access Denied / 403

检查 access-key 和 secret-key 是否与启动容器时设置的一致。

如果改过密码，需要在控制台确认。

❌ The specified bucket does not exist

如果你删除了 @PostConstruct 中自动建桶的逻辑，需要在控制台手动创建 Bucket。

桶名必须和 application.yml 中配置的一致。

❌ OkHttp / Kotlin 依赖冲突

在 MinIO 依赖中排除 OkHttp（见第 ② 步）。

或者在 <dependencyManagement> 中统一 OkHttp 版本为 4.12.0。

⚠️ 上传大文件超时

在 application.yml 中增大上传限制：

yaml application.yml 
spring:
  servlet:
    multipart:
      max-file-size: 100MB
      max-request-size: 100MB
💡 如何让上传的图片可以直接通过 URL 在浏览器预览？

需要将 Bucket 的访问策略设为 public。在 MinIO 控制台 → 选择桶 → Access Policy 改为 public。

这样 http://localhost:9000/weixiu/xxx.jpg 就可以直接在浏览器打开了。

如果不想设为 public，使用预签名 URL（/api/file/presigned 接口）来生成临时访问链接。
