# 案例/经验知识沉淀 — 前端实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: 用 superpowers:subagent-driven-development 或 superpowers:executing-plans 逐任务实现。步骤用 `- [ ]` 勾选跟踪。
> **验证方式**：本项目前端无单测，验证统一用 `npm run build` 通过 + dev 热更人工走查（登录态/任务状态前置）。**只提交 `src/` 源文件，不提交 `dist/` 构建产物**。

**Goal:** 工人在已完成任务上「沉淀为案例」→ AI 草稿可编辑 → 提交（合规闸门拦截会显示理由）；管理员在「沉淀审核」内审核（可修正）通过/驳回；案例可在图谱故障下查看。

**Architecture:** 一个可复用的「案例草稿编辑弹窗」组件承载所有通道（期1任务，期2/3 文件/语音复用）。工人侧入口在任务详情，管理员侧入口并入现有审核页签。

**Tech Stack:** Vue3 `<script setup>` + Element Plus + 现有 `src/api/request.js`、`src/api/graph.js`。

参考 spec：`docs/superpowers/specs/2026-06-09-案例经验知识沉淀-design.md`；后端计划：`2026-06-09-案例沉淀-后端Java与Python计划.md`

## 期 1

### Task 1: api 封装

**Files:** Create `src/api/caseRecord.js`

- [ ] **Step 1: 写接口**
```js
import { request } from './request'
export const draftFromTask = (taskId) => request({ url:`/weixiu/case-record/draft-from-task/${taskId}`, method:'POST' })
export const submitCase   = (data)   => request({ url:'/weixiu/case-record/submit', method:'POST', data, throwOnError:true })
export const getPendingCases = (page=1,size=10) => request({ url:'/weixiu/case-record/pending', method:'GET', params:{page,size} })
export const approveCase  = (id,data)=> request({ url:`/weixiu/case-record/${id}/approve`, method:'POST', data, throwOnError:true })
export const rejectCase   = (id,comment)=> request({ url:`/weixiu/case-record/${id}/reject`, method:'POST', params:{comment}, throwOnError:true })
export const getMyCases   = (page=1,size=10)=> request({ url:'/weixiu/case-record/mine', method:'GET', params:{page,size} })
```
- [ ] **Step 2: build 通过 + Commit**
```bash
npm run build 2>&1 | tail -2
git add src/api/caseRecord.js && git commit -m "feat(case): 前端案例接口封装"
```

### Task 2: 案例草稿可编辑弹窗组件

**Files:** Create `src/components/case/CaseSubmitDialog.vue`

- [ ] **Step 1: 组件**

`props: { visible, draft }`(draft = 后端 CaseDraftVO)；`emits: ['update:visible','submitted']`。
- 用 `el-dialog` + 表单：title/summary/diagnosis/resolution/result/experienceSummary/tags/downtime/cost；图片预览(draft.imageUrls)。
- 设备项：`el-select`(可空，数据源 `searchDevices('')`)，默认选中 `draft.deviceId`（来源任务带入）。
- 「提交」→ 组装 dto(含 sourceType=draft来源, sourceTaskId, deviceId, faultName, imageUrls + 全字段)→ `submitCase(dto)`。
- **合规拦截**：submit 抛错时(throwOnError)，用 `ElMessage.error(err.message)` 展示后端返回的合规 reason，弹窗不关、让工人改。
- 成功 → `ElMessage.success('已提交，等待审核')` → emit submitted + 关闭。

- [ ] **Step 2: build + Commit**
```bash
npm run build 2>&1 | tail -2
git add src/components/case/CaseSubmitDialog.vue && git commit -m "feat(case): 案例草稿可编辑提交弹窗"
```

### Task 3: 工人侧入口(任务详情「沉淀为案例」)

**Files:** Modify `src/views/userViews/UserTaskDetail.vue`

- [ ] **Step 1: 加按钮 + 拉草稿**

- 顺带加「我的案例」按钮/抽屉，调 `getMyCases` 列出状态(pending/approved/rejected + reviewComment)。

### Task 4: 管理员审核子页

**Files:** Create `src/components/case/CaseReviewPanel.vue`；Modify 现有审核宿主页(如 `src/views/adminViews/AdminKnowledge.vue`)

- [ ] **Step 1: CaseReviewPanel.vue**
- `getPendingCases` 列出待审；点一条进详情(右侧/弹窗)。
- 详情**所有字段可编辑**(标注/修正大模型输出)：title…experienceSummary/tags/deviceId/faultName。
- 「通过」→ `approveCase(id, 编辑后dto)`；**向量化失败时后端报错** → `ElMessage.error` 提示可重试(不关面板)。
- 「驳回」→ 弹输入意见 → `rejectCase(id, comment)`。

走查：工人提交后 → 管理员「案例审核」看到待审 → 改一处 → 通过 → 列表移除；再造一条驳回带意见。
