百度语音识别（ASR）接口流程

VoiceToTextUtils.java — 从音频上传到文字返回的完整链路

调用流程图

1. 前端上传音频文件
POST /weixiu/ai/transcribe (MultipartFile)
AiController.transcribeByBaiDu()
2. Service 层委托
直接转发给工具类处理
AiServiceImpl.getStringByVoiceViaBaidu()
3. 校验音频文件
检查：非空 → 大小 ≤ 10MB → 后缀白名单
(mp3/wav/webm/m4a/mp4/ogg/amr/pcm)
validateFile()
校验通过
4. 写入临时文件
MultipartFile → 临时文件 upload-asr-xxx.{ext}
Files.createTempFile() + Files.copy()
5. FFmpeg 转码为 PCM
任意音频 → PCM 16-bit 单声道 16kHz
超时保护：30秒未完成则强制终止
convertToPcm()
6. 获取百度 Access Token
优先使用缓存 Token（提前1小时刷新）
POST https://aip.baidubce.com/oauth/2.0/token
grant_type=client_credentials
getAccessToken()
7. 调用百度 ASR 接口
POST https://vop.baidu.com/server_api
Content-Type: audio/pcm;rate=16000
Body: PCM 字节流
asrClient.post() → BaiDuAsrDTO
err_no == 0 ?
✓ err_no == 0
8a. 返回识别文本
result 数组拼接为完整字符串
String.join("", result)
✗ err_no != 0
8b. 抛出异常
"百度语音识别失败：" + errMsg
9. finally：清理临时文件
删除 upload-asr-xxx.{ext} 和 baidu-asr-xxx.pcm
deleteQuietly(inputFile) + deleteQuietly(pcmFile)

涉及的类

VoiceToTextUtils

ai.weixiu.utils

核心工具类，包含音频转码、Token 管理、ASR 调用的完整逻辑。持有两个 WebClient 实例分别访问不同域名。

  • tokenClient — aip.baidubce.com
  • asrClient — vop.baidu.com
  • cachedToken — Token 缓存
  • expireAtMillis — Token 过期时间戳

BaiDuConfigurationProperties

ai.weixiu.entity

配置类，绑定 application.yml 中 baidu.speech.* 前缀。

  • apiKey / secretKey — OAuth 凭据
  • cuid — 设备标识
  • devPid — 语言模型ID（1537=普通话）
  • rate — 采样率（默认16000）
  • maxUploadMb — 上传大小限制

BaiDuAsrDTO

ai.weixiu.pojo.dto

百度 ASR 接口响应体映射。

  • errNo — 错误码（0=成功）
  • errMsg — 错误信息
  • sn — 唯一序列号
  • result — 识别结果列表

BaiDuTokenDTO

ai.weixiu.pojo.dto

百度 OAuth 2.0 Token 响应体映射。

  • accessToken — 访问令牌
  • expiresIn — 有效期（秒）

方法详解 — VoiceToTextUtils

transcribe public
public String transcribe(MultipartFile file)
主入口方法，编排整个语音识别流程：
  • 调用 validateFile() 校验文件合法性
  • 将 MultipartFile 写入临时文件
  • 调用 convertToPcm() 转码为 PCM 格式
  • 调用 getAccessToken() 获取 Token
  • 通过 asrClient 发送 PCM 数据到百度 ASR
  • 解析响应，拼接识别结果返回
  • finally 块无论成功失败都清理临时文件
入口 HTTP 超时 15s 自动清理临时文件
convertToPcm public
public Path convertToPcm(Path inputFile)
音频转码，调用系统 FFmpeg 将任意格式音频转为百度要求的 PCM 格式：
  • 编码：PCM 16-bit 有符号小端（pcm_s16le）
  • 声道：单声道（-ac 1）
  • 采样率：16000 Hz（-ar 16000）
  • 剥离视频轨（-vn）只取第一条音轨（-map 0:a:0）
  • 合并 stderr 到 stdout 防止管道阻塞
  • 30 秒超时，超时自动 destroyForcibly()
FFmpeg 超时 30s 返回临时 .pcm 文件路径
getAccessToken public synchronized
public synchronized String getAccessToken()
OAuth 2.0 Token 管理，带缓存和线程安全：
  • 优先返回缓存 Token（未过期时直接返回）
  • Token 过期前 1 小时主动刷新（expiresIn - 3600）
  • 通过 tokenClient 请求 aip.baidubce.com
  • synchronized 保证多线程下只发一次请求
  • 默认有效期 30 天（2592000 秒）
线程安全 Token 缓存 HTTP 超时 15s
validateFile private
private void validateFile(MultipartFile file)
文件校验，三重检查：
  • 非空检查：file != null && !file.isEmpty()
  • 大小检查：不超过 maxUploadMb（默认 10MB）
  • 格式白名单：mp3 / wav / webm / m4a / mp4 / ogg / amr / pcm
校验失败抛出 IllegalArgumentException。
参数校验 NPE 防护
getSuffix private
private String getSuffix(String filename)
提取文件后缀名（含点号），用于创建临时文件和格式校验。
  • 输入 "recording.mp3" → 返回 ".mp3"
  • 输入 null 或无点号 → 返回 ".audio"（兜底）
工具方法
deleteQuietly private
private void deleteQuietly(Path path)
静默删除临时文件，即使删除失败也不抛异常。
  • null 安全：path 为 null 直接返回
  • 用于 finally 块，确保不影响正常返回值或已有异常
资源清理 异常吞没

完整调用链

// Controller
AiController.transcribeByBaiDu(MultipartFile)
  ↓
// Service
AiServiceImpl.getStringByVoiceViaBaidu(MultipartFile)
  ↓
// 工具类主方法
VoiceToTextUtils.transcribe(MultipartFile)
  ├─ validateFile(file)  // 校验
  ├─ convertToPcm(inputFile)  // FFmpeg 转码
  ├─ getAccessToken()  // OAuth Token（带缓存）
  ├─ asrClient.post()  // 百度 ASR 识别
  └─ deleteQuietly() × 2  // finally 清理
